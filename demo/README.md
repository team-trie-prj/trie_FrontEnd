# 지능형 정보 시스템 · 융합 데모

**생성형 AI 기반 지능형 정보 시스템** — 두 파트(① VLM 비전, ② 하이브리드 RAG/에이전트)를
하나의 **AI 에이전트 허브**로 통합한 단일 웹앱입니다. 제출하신 **융합 와이어프레임의 8개 화면**을
그대로 구현했고, **Material Design 2 색상 시스템**과 **직접 그린 SVG 아이콘 세트**를 적용했으며,
**ATDD(인수 테스트)** 로 제안서의 모든 기능을 검증합니다.

> 자연어 질의 → AI 에이전트 → 도구 호출(VLM·YOLO / 하이브리드 RAG / 공공데이터) → 검색·요약·보고서·업무자동화

---

## 1. 임시 데모 시연 (가장 빠른 경로)

**Windows**: `run.cmd` 더블클릭  ·  **PowerShell**: `./run.ps1`  ·  **mac/Linux**: `./run.sh`

수동 실행:
```bash
pip install -r requirements.txt      # 최초 1회 (모두 GPU 불필요)
python app.py                        # → http://localhost:8000
```

- **API 키·GPU 없이 끝까지 동작**합니다(오프라인 폴백). `rag.db`(263건)는 동봉되어 즉시 시작됩니다.
- 키를 넣으면(`.env`) 자동 승격: `GEMINI_API_KEY`(에이전트·VLM), `VOYAGE_API_KEY`(임베딩), `DATAGO_SERVICE_KEY`(공공데이터).

### 발표용 시연 시나리오 (제안서 4대 시나리오)

1. **메인** 화면에서 `대전 유성구 포트홀 영역을 찾아줘` 입력 → **질의**
2. **자연어 질의** 화면 — 에이전트가 *의도 분석 → VLM/YOLO 탐지 → 하이브리드 RAG 검색 → 공공데이터 집계 → AI 응답* 순서로
   **어떤 도구를 호출했는지 투명하게** 보여줍니다. (제안서 차별점: 신뢰성·검증성)
3. **이미지 분석** — `샘플 불러오기` → 모델(YOLO/Gemini/Mock) 선택 → **탐지 실행** → bbox·심각도·신뢰도·COCO 메타데이터
4. **공공데이터 연계** — 자치구 지도 공간조인 + 자치구 랭킹 + 대기질
5. **보고서** — 포함 섹션 선택 → **.docx 생성**(미리보기 동기화)
6. **데이터 관리** — 검수(승인/반려) → 변경 이력 자동 기록 · `프롬프트·응답` 탭에서 질의 이력
7. **업무 자동화** — 4단계 절차 자동 추천 → **절차 실행**(탐지·통계·보고서 오케스트레이션)

## 2. 디자인 시스템 (업로드 디자인 적용)

업로드하신 다크 테마 디자인을 그대로 적용했습니다.
- **테마**: 블랙(#000) 배경 · Pretendard 폰트(로컬 임베드 `web/fonts/`) · Material Symbols 아이콘 · 흰색 포인트 · 가벼운(300) 대형 헤딩.
- **헤더 네비게이션**: 계열별로 묶은 **호버 드롭다운** — 메인 · 분석(자연어 질의·이미지 분석) · 데이터(검색·공공데이터·데이터 관리) · 산출(보고서·업무 자동화). 마우스를 올리면 하위 항목이 펼쳐집니다.
- **푸터**: 모든 페이지 공통 푸터(브랜드·분석·데이터·산출·시스템 링크 + 저작권·기술 배지)를 구성했습니다.
- 폰트는 로컬 임베드라 **오프라인에서도 디자인이 그대로** 표시됩니다.

## 3. 통합 아키텍처 (PM/통합 담당)

```
  web/index.html (단일 통합 셸 · 8개 화면)
        │  REST (/api/*)
        ▼
  app.py  =  AI 에이전트 오케스트레이션 허브   (표준 라이브러리, 의존성 0)
        ├──────────────► vlm_bridge.py  (파트 A)  → VLM/vlm  (실제 패키지 ↔ mock 폴백)
        └──────────────► Hybrid_RAG/src (파트 B)  → agent · retrieve · report · review
                              │
                     rag.db (UnifiedDoc 263건 · 임베딩/BM25)
```

두 코드베이스는 **수정 없이** 그대로 두고, `app.py`(허브) + `vlm_bridge.py`(접합부) 두 파일로 통합했습니다.
한 번의 질의에 에이전트가 **비전 탐지 → RAG 검색 → 공공 통계**를 순서대로 호출합니다.

### 8개 화면 ↔ 백엔드
| 화면 | 사용 코드(통합) |
|---|---|
| A 메인 | `/api/overview` |
| B 자연어 질의(추론) | `agent.analyze_intent` + 도구 오케스트레이션 |
| C 이미지 분석 | `vlm_bridge → VLM/vlm` (`/api/detect`) |
| D 검색 결과 | `retrieve.hybrid_search` |
| E 공공데이터 연계 | `geo_payload` 공간조인 + 공공 CSV |
| F 보고서 | `report.generate_report` (.docx) |
| G 데이터 관리 | `review.py` + `doc_change_log` |
| H 업무 자동화 | `automation_payload` 오케스트레이션 |

## 4. ATDD 인수 테스트

제안서가 명시한 모든 기능을 **수용 기준(Given/When/Then)** 으로 정의하고 실행 검증합니다.

```bash
python tests/acceptance/acceptance.py
```
- 통합 서버를 임시 포트로 자동 기동 → 16개 시나리오 실행 → 종료(외부 의존성 0).
- 종료코드 0=전체 통과.

| ID | 검증 기능(제안서) |
|---|---|
| AC-01 | 자연어 질의 입력 |
| AC-02 | 에이전트 추론 투명성(의도→도구→응답) |
| AC-03 | 이미지 자동 이해·라벨링(VLM, bbox·심각도·COCO) |
| AC-04 | 탐지 모델 선택(YOLO 파인튜닝/VLM/mock) |
| AC-05 | 하이브리드 검색(의미+키워드 융합·재정렬) |
| AC-06 | 공공데이터 연계(지역 공간조인·대기질) |
| AC-07 | 통계 집계/순위 |
| AC-08 | 검색 결과 요약 |
| AC-09 | 보고서 자동 생성(.docx) |
| AC-10 | 데이터 변경 이력 관리 |
| AC-11 | 검수 상태 변경 → 이력 자동 기록 |
| AC-12 | 에이전트 기반 업무 자동화 |
| AC-13 | 프롬프트·응답 이력 |
| AC-14 | 통합 데이터 구조(UnifiedDoc) |
| AC-15 | 제안서 4대 시나리오 흐름 |
| AC-16 | UI 8개 화면 + 커스텀 SVG 아이콘 + MD2 |

## 5. 폴더 구조
```
integrated_demo/
├─ app.py                 # 통합 서버 = 에이전트 허브 (진입점)
├─ vlm_bridge.py          # 비전 통합 브리지 (파트 A 접합부)
├─ web/index.html         # 단일 통합 셸 · 8개 화면 · MD2 · 커스텀 SVG 아이콘
├─ data/samples/          # 데모용 도로 이미지
├─ tests/acceptance/      # ATDD 인수 테스트
├─ Hybrid_RAG/  VLM/      # 파트 A·B 원본 코드베이스 (그대로)
├─ requirements.txt · run.cmd · run.ps1 · run.sh · .env.example
```

## 6. 동작 모드 (전부 폴백 내장 — 데모 안정성 우선)
- **비전**: 실제 VLM 패키지(`pip install` 후) → 없으면 내장 mock.
- **에이전트**: `GEMINI_API_KEY` → 없으면 규칙 라우터.
- **임베딩**: `VOYAGE_API_KEY` → 없으면 로컬 해시.
- 어떤 경우에도 8개 화면·16개 인수 기준이 모두 동작합니다.
