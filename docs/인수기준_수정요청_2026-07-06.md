## 2. `03_search.feature` — 첨부 이미지 제약 Scenario Outline 교체

`@FNC-SRC-01 ... Scenario Outline: 첨부 이미지 제약 위반 시 거부` 전체를 아래로 교체 제안:

```gherkin
@FNC-SRC-01 @REQ-SRCH @negative @edge @ui
Scenario Outline: 첨부 이미지 제약 처리 (클라이언트 리사이징 정책)
  Given 사용자가 로그인되어 있다
  When 사용자가 "<용량>" 크기의 "<확장자>" 이미지를 첨부한다
  Then 시스템은 <처리>

  Examples:
    | 용량             | 확장자 | 처리                                  |
    | 6MB             | JPG  | 클라이언트에서 리사이징 후 5MB 이하로 축소하여 허용한다   |
    | 8MB             | PNG  | 클라이언트에서 리사이징 후 5MB 이하로 축소하여 허용한다   |
    | 리사이징 후에도 5MB 초과 | JPG  | 거부하고 용량 초과 사유를 안내한다               |
    | 31MB            | JPG  | 즉시 거부한다 (리사이징 불가 상한 30MB 초과)       |
    | 3MB             | GIF  | 거부하고 사유(허용되지 않은 확장자, JPG/PNG만)를 안내한다 |
    | 2MB             | BMP  | 거부하고 사유(허용되지 않은 확장자, JPG/PNG만)를 안내한다 |
```
