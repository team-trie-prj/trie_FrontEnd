# 프론트엔드 개발 현황 및 팀 확인 요청

작성: 김성민 · 2026-07-02

## 현황 요약

기능 명세서 11개 기능(FNC-AUTH-01 ~ FNC-HIS-01)의 프론트엔드 구현이 완료되었습니다.
백엔드 API 명세 미확정으로 현재 mock 모드로 전체 플로우가 동작하며,
연동 시 `src/api/endpoints.ts` 경로 교체 + `.env` 전환만 하면 됩니다.

- 화면: 로그인 / 통합검색 / 검색결과 / 데이터관리(적재 현황 포함) / API관리 / 보고서 뷰어 / 히스토리 사이드바
- 품질: 단위 테스트 21건, Lighthouse 성능·접근성·SEO 개선, 반응형(모바일) 대응
- 보고서: TipTap 위지윅 편집(서식 툴바), PDF·DOCX 파일 내보내기

## 확인 요청 ① 폐쇄망 배포 관련 (긴급)

인터넷 불가 환경 배포로 확인되어 폰트·아이콘은 번들 포함(self-host)으로 조치 완료했습니다.
다만 **카카오 로그인(FNC-AUTH-01)은 kauth.kakao.com 접근이 필수라 폐쇄망에서 동작 불가**합니다.

- 질문: 인증을 내부 계정(ID/PW) 방식으로 변경할지, 망연계 구간을 통해 카카오 인증을 허용할지 결정 필요
- 같은 이유로 공공데이터포털 호출도 백엔드 서버의 외부 통신 경로 확인 필요

## 확인 요청 ② 명세 개정 안건 2건

1. **검색 결과 AI 종합 답변**: 데모에는 있었으나 명세(FNC-VIW-01)에는 없어 미구현 상태로 유지 중.
   구현 코드는 준비되어 있어(revert 커밋 490a9f9) 명세 반영 결정 시 즉시 복원 가능.
   백엔드는 검색 응답에 `answer` 필드 추가 필요.
2. **공공데이터포털 API 검색(탐색)**: 명세(FNC-PUB-01)는 수동 등록만 정의.
   포털 목록 검색 연동을 원하면 명세 추가 + 백엔드 프록시 필요 (폐쇄망이면 불가).

## 확인 요청 ③ WBS 갱신

- 기존 17개 항목: 프론트 구현 완료 → 현황 갱신 요청
- 누락 항목 8건 추가 요청: 문서 업로드/처리 상태 UI, API 카탈로그 등록·목록, VLM 폴백 모달,
  토큰 재발급, DOCX 내보내기, 히스토리 목록 (상세 표는 별도 공유)

## 백엔드에 필요한 것 (연동 시)

- REST API 명세서 (endpoints.ts placeholder 참조: auth/search/documents/catalog/reports/history)
- 보고서 스트리밍 포맷(SSE 여부), 보고서 편집본 서버 저장 API 여부
- Refresh Token 전달 방식 (HttpOnly 쿠키 권장)
- 검색 응답 hits 최대 10개 제한(FNC-SRC-03) 및 출처 메타데이터(provenance) 필드 구조 합의
