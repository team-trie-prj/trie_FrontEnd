## 총평

FE 담당 시나리오에서 **결함 0건**. 다만 인수 기준과 기존 문서(기능명세서·WBS)가 상충하는 항목 2건이 발견되어 협의가 필요하다(아래 "협의 필요" 참조). 상충 2건은 테스트 코드에 `it.fails`(expected-fail)로 남겨 두어, 협의 결과에 따라 코드를 바꾸면 테스트가 즉시 이를 감지한다.

| VLM 지연 시 텍스트 단독 전환 확인 | ✅ | 타임아웃 → 전환 확인 상태 → 동의 시 이미지 제외 검색 자동 검증 |

| [캐시 차단·클러스터] 노드 분산 반복 요청 | ⏳ | 서버 캐시 계층은 BE/Ops. 클라이언트 몫(고유 세션 헤더)은 위에서 검증 |

## 🟡 협의 필요 (2건)

1. **RBAC — 실무자의 데이터 관리 접근**: 인수 기준은 차단, 기능명세서는 "관리자 또는 실무자" 허용. 어느 쪽이 맞는지 결정 필요. 차단으로 확정되면 `App.tsx`의 roles를 `['admin']`으로 한 줄 수정 + `auth.rbac` expected-fail 해제.
2. **이미지 5MB 초과 처리**: 인수 기준은 6MB부터 즉시 거부, 현 구현은 WBS 기반 클라이언트 리사이징 후 5MB 재검증(현장 고화질 사진 UX 개선 목적). 즉시 거부로 확정되면 리사이징 제거 또는 "리사이징 후에도 5MB 초과 시 거부"(현행) 중 기준 명문화 필요.
